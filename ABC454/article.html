ABC454 题解：图论深搜与栈的巧妙运用

一、为什么选这两道题来写

在 AtCoder Beginner Contest 的题单里，C 题和 D 题是一道隐形的分水岭。

A、B 两题通常只需要基础语法和简单模拟，只要学过编程就能做。但从 C 题开始，题目会要求你建立抽象的数学模型，或者灵活运用基础数据结构。如果这套思路没有打通，后面的 E、F 题基本无从谈起。

这次 ABC454 的 C 题和 D 题，恰好分别对应了算法竞赛中最核心的两个基础能力：

  • C 题 — 有向图上的深度优先搜索（DFS）：将物品交换关系建模为有向图，问题转化为从起点出发能到达多少节点。
  • D 题 — 栈的灵活运用：字符串的局部替换规则，可以通过栈在 O(N) 时间内完成规范化，进而判断两个字符串是否等价。

图论和数据结构之所以重要，是因为它们处于整个算法知识体系的承上启下位置。向上，它们连接着数组、链表、递归这些最基础的概念；向下，它们延伸出最短路径、拓扑排序、并查集、线段树等一系列进阶工具。能把这两道题吃透，说明你已经从"会写代码"跨入了"会设计算法"的阶段。

下面进入具体的题目分析。

二、C 题：Straw Millionaire（有向图 + DFS）

题意

有 N 种物品，编号 1 到 N。高桥君最初只有物品 1。

他有 M 个朋友。第 i 个朋友有一个交换规则：如果把物品 A_i 给他，就能换得物品 B_i。

问：高桥君最终能获得多少种不同的物品（包括初始的物品 1）？

关键观察

这道题的关键在于把交换规则抽象成有向图：

  • 每种物品是一个节点。
  • 每个交换规则 "A_i → B_i" 是一条从 A_i 指向 B_i 的有向边。

建图之后，问题就变得很直观了：从节点 1 出发，沿着有向边能走到哪些节点？这些节点的总数就是答案。

因为有向边是有方向的（只能 A→B，不能 B→A），所以不能用并查集，必须用图的遍历算法。数据范围是 N, M ≤ 3×10^5，DFS 或 BFS 都可以轻松通过。

图中红色节点是起点（物品 1），蓝色节点是可到达的物品。从 1 出发沿着有向边走，能到达 1、2、3、4 共 4 个节点。

代码解析

adj[u] 存储从节点 u 出发的所有有向边。vis[u] 标记节点 u 是否已被访问过。

dfs(u) 函数递归遍历：如果 u 已访问则直接返回；否则标记为已访问，然后递归访问 u 的所有邻居 v。这里递归深度最多是 N = 3×10^5，在 C++ 中需要确保栈空间足够（通常默认栈空间可以承受）。

主函数先读入 N 和 M，然后建图。注意题目输入的节点编号是从 1 开始的，所以直接按原编号存入 adj 即可，无需偏移。

最后从节点 1 开始 DFS，统计所有 vis[i] == true 的节点数量并输出。

三、D 题：(xx)（栈规范化）

题意

给定一个只包含 (, x, ) 的字符串 A。你可以进行两种操作任意次数：

  • 把子串 (xx) 替换成 xx
  • 把子串 xx 替换成 (xx)

再给定一个同样只包含 (, x, ) 的字符串 B。问：是否能把 A 变成 B？

多组测试用例。

关键观察

这道题最容易卡住的地方，是觉得要在 A 和 B 之间找一系列变换步骤。但其实有一个更巧妙的视角：两种操作互为逆操作。

这意味着，如果 A 能变成 B，那么 B 也能变成 A。进一步想：所有能互相转换的字符串，其实都属于同一个"等价类"。我们只要找到每个等价类的唯一代表元（规范形式），然后比较 A 和 B 的代表元是否相同即可。

什么是最简的规范形式？就是字符串中不再含有 (xx) 子串的形式。因为只要有 (xx)，就可以用第一种操作把它缩短成 xx。

如何从任意字符串得到规范形式？从左到右扫描，用一个栈维护当前结果：

  • 每读入一个字符，就把它压入栈
  • 如果栈顶四个字符恰好是 (xx)，就把这四个字符弹出，压入 xx

这个过程类似于括号匹配的栈处理，时间复杂度是线性的 O(|S|)。

当栈顶出现 (xx) 时，立即将其替换为 xx。最终得到规范形式 xxx。

代码

代码解析

f(s) 函数是核心。它用一个 vector<char> 当作栈，逐个字符扫描输入字符串。

关键判断在 if 语句：当前字符是 )，且栈中已有至少 3 个字符，栈顶三个字符恰好是 x, x, (（注意栈顶是最后一个元素，所以顺序是 st[n-1]='x', st[n-2]='x', st[n-3]='('），说明形成了 (xx) 模式。

此时，把 ( 的位置改成 x（st[n-3]='x'），然后弹掉栈顶的一个 x（st.pop_back()）。这样栈里原来的 (, x, x, ) 就变成了 x, x，正好对应替换规则。

如果不满足替换条件，就把当前字符正常压入栈。

solve() 函数读入两个字符串，分别求规范形式，然后比较是否相等。main() 处理多组测试用例。

复杂度分析

每个字符最多入栈一次、出栈一次，所以单次简化的复杂度是 O(|S|)。所有测试用例的 |A|+|B| 总和不超过 2×10^6，完全可以轻松通过。

四、写在最后

C 题告诉我们：建模是解题的第一步。把物品交换看成有向图之后，问题就从"怎么换"变成了"能走到哪"，思路瞬间清晰。

D 题告诉我们：观察操作的可逆性往往能打开突破口。两种操作互为逆操作，意味着存在规范形式；而规范形式又恰好可以用栈在线性时间内求出。

这两道题的共同点是——它们都不需要高深的算法模板，但对问题建模的敏感度有很高要求。而这种敏感度，正是从一次次实战中磨出来的。
